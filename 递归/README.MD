# 递归

> 递归（Recursion）是一种函数调用自身的编程技巧。核心思想：**将大问题分解为结构相同的子问题，直到达到基本情况（base case）**。

## 核心要素

1. **基本情况（Base Case）**：递归的终止条件，不再调用自身
2. **递归步骤（Recursive Step）**：将问题规模缩小，向基本情况靠近

## 文件说明

| 文件 | 说明 |
|------|------|
| `课程中的示例递归.py` | 演示递归的执行顺序 — "先打印再下潜" vs "先下潜再打印" |
| `quicksort.py` | 快速排序 — 分治法：选基准，分左右，递归排序 |
| `inorder.py` | 二叉树中序遍历 — 递归天然匹配树的嵌套结构 |
| `permute.py` | 全排列 — 递归 + 回溯：做选择、递归、撤销选择 |
| `hanoi.py` | 汉诺塔 — 将 n 个盘子分解为 n-1 个盘子的子问题 |

## 理解递归的关键

- **信任递归（Leap of Faith）**：假设函数能正确处理子问题，不要陷入层层展开
- **递归 = 递（向下分解）+ 归（向上返回）**
- 打印语句在递归调用**之前**还是**之后**，决定了"下潜时做事"还是"上浮时做事"
